  ---
  Request flows

  Upload flow

  Client → main.py → parser.py (decode + clean DF)
                   → session_store.py (create session, store DFs)
                   → schema_analyzer.py (stats + starter questions)
                   ← UploadResponse (session_id, schema, questions)

  Query flow

  Client → main.py → session_store.py (get session + DFs)
                   → claude_service.py (system prompt + history → Bedrock)
                   → code_executor.py (AST check → subprocess → result)
                   → _inject_result_value() (replace RESULT_VALUE placeholder)
                   → RESULT_DATA Injection (inject JSON into HTML artifact)
                   → session_store.py (append to history)
                   ← QueryResponse (chat_message, artifact HTML, insight)

  ---
  Key design decisions

  ┌─────────────────────────────┬────────────────────────────────────────────────────────────────────┐
  │          Decision           │                               Reason                               │
  ├─────────────────────────────┼────────────────────────────────────────────────────────────────────┤
  │ In-memory caching/LRU       │ MVP blistering read/write speed for EDA; gracefully pages to disk  │
  ├─────────────────────────────┼────────────────────────────────────────────────────────────────────┤
  │ Subprocess execution        │ Isolates user computation from web server; fast & avoids Docker lag│
  ├─────────────────────────────┼────────────────────────────────────────────────────────────────────┤
  │ AST validation before exec  │ Drops OS/sys/subprocess hacking instantly before spawned execution │
  ├─────────────────────────────┼────────────────────────────────────────────────────────────────────┤
  │ RESULT_DATA & RESULT_VALUE  │ Python natively intercepts & injects math, preventing LLM falsehood│
  ├─────────────────────────────┼────────────────────────────────────────────────────────────────────┤
  │ Schema injected into prompt │ Claude knows column names, types, stats without seeing raw PII data│
  ├─────────────────────────────┼────────────────────────────────────────────────────────────────────┤
  │ Client-side PDF Export      │ Bypasses complex Puppeteer servers by snapshotting DOM natively    │
  └─────────────────────────────┴────────────────────────────────────────────────────────────────────┘
